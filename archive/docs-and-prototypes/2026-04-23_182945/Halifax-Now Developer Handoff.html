The Project

§ 01

Halifax Now is an event discovery site for Halifax, Nova Scotia. It aggregates events from multiple sources via The Events Calendar Pro + Event Aggregator on WordPress. The frontend is a custom theme (or headless React app) that renders events in the Broadsheet design direction — a hybrid of editorial magazine discipline and gig-poster energy.

The design direction.

The active design is called Broadsheet. It combines:

• Editorial shell — Playfair Display serif masthead, Source Serif 4 body, critic's picks voice, NY Mag–style blurbs

• Poster energy — Anton display type for headlines, acid-yellow (#e8ff00) accents, black drop-shadows on cards, halftone image treatment, bold section banners

Image Treatments

§ 04

Every event image is normalized through a CSS filter + blend-mode stack. Submitters upload anything — the treatment makes it look like it belongs. No server-side processing. No canvas. Pure CSS layers.

The core principle.

Image sits as background-image on a positioned container. Two absolutely-positioned overlay divs stack on top with mix-blend-mode. The hue value (0–360) comes from the event's primary category and is the only variable per event.

Treatment 1 — Halftone

Used on the Broadsheet homepage, event cards, event detail hero, venue hero, map popup. This is the primary treatment.

/* Container */
.event-image {
  position: relative;
  overflow: hidden;
  background: #000;
}

/* Layer 1: the photo — grayscale + contrast boost */
.event-image__photo {
  position: absolute; inset: 0;
  background-image: url(photo.jpg);
  background-size: cover;
  background-position: center;
  filter: grayscale(1) contrast(1.35) brightness(0.95);
}

/* Layer 2: category colour tint */
.event-image__tint {
  position: absolute; inset: 0;
  background: oklch(62% 0.22 {HUE}); /* HUE from category */
  mix-blend-mode: multiply;
}

/* Layer 3: halftone dot pattern */
.event-image__dots {
  position: absolute; inset: 0;
  background: radial-gradient(circle at 1px 1px, rgba(0,0,0,0.85) 1px, transparent 1.4px) 0 0 / 5px 5px;
  mix-blend-mode: multiply;
  opacity: 0.55;
}

Treatment 2 — Duotone

Used in the Listings/NY Mag variant. Softer, more editorial.

/* Layer 1: soft sepia photo */
.event-image__photo {
  filter: grayscale(1) contrast(1.05) brightness(1.05) sepia(0.15);
  mix-blend-mode: multiply;
}
/* Layer 2: warm gradient tint */
.event-image__tint {
  background: linear-gradient(160deg,
    oklch(92% 0.04 {HUE}) 0%,
    oklch(78% 0.08 {HUE}) 100%
  );
  mix-blend-mode: multiply;
  opacity: 0.85;
}

Treatment 3 — Neon

Used in the Tonight Board variant. Dark and moody.

/* Layer 1: dark photo */
.event-image__photo { filter: grayscale(1) contrast(1.3) brightness(0.55); }
/* Layer 2: radial colour bloom */
.event-image__tint {
  background: radial-gradient(circle at 70% 30%, oklch(65% 0.28 {HUE}) 0%, transparent 55%);
  mix-blend-mode: screen; opacity: 0.9;
}
/* Layer 3: bottom vignette */
.event-image__vignette {
  background: linear-gradient(180deg, transparent 40%, rgba(0,0,0,0.6) 100%);
}

Shadow Boxes

§ 05

Cards, buttons, and containers use an offset drop shadow — not a CSS box-shadow blur, but a hard offset using a solid color. This is the poster/print-inspired detail that makes the design feel physical.

/* Standard card */
.card {
  border: 2.5px solid var(--ink);
  box-shadow: 5px 5px 0 var(--ink);  /* hard offset, no blur */
  background: var(--paper);
}
.card:hover {
  transform: translate(-2px, -2px);
  box-shadow: 7px 7px 0 var(--ink);  /* offset increases on hover */
}

/* CTA button */
.cta-btn {
  border: 3px solid var(--ink);
  box-shadow: 6px 6px 0 var(--ink);
  background: var(--red); color: var(--paper);
}
.cta-btn:hover {
  transform: translate(-2px, -2px);
  box-shadow: 9px 9px 0 var(--ink);
}
.cta-btn:active {
  transform: translate(4px, 4px);
  box-shadow: 0 0 0 var(--ink);  /* presses in on click */
}

/* Small chip / tag */
.chip {
  border: 2px solid var(--ink);
  box-shadow: 2px 2px 0 var(--ink);
}
.chip.on {
  transform: translate(2px, 2px);
  box-shadow: 0 0 0 var(--ink);  /* selected state depresses */
}

The Data Contract

§ 08

The front-end expects a flat JSON shape per event. Build a REST endpoint at /wp-json/hfx/v1/events that returns this. Nullable fields are optional.

{
  id:          "e1",                    // WP post ID (required)
  title:       "Halifax Jazz Fest",      // post_title (required)
  short:       "One-liner for cards",    // hfx_short_blurb ACF
  blurb:       "2–3 sentence editorial", // hfx_editor_blurb ACF
  date:        "2026-04-24",            // _EventStartDate (required)
  time:        "20:00",                 // _EventStartDate time part
  endTime:     "23:30",                 // _EventEndDate time part
  venue:       "The Carleton",          // tribe_venue name (required)
  address:     "1685 Argyle St",        // tribe_venue address
  hood:        "Downtown",              // hfx_neighbourhood ACF
  category:    "music",                 // tribe_events_cat slug (required)
  mood:        ["chill","crew","date"], // hfx_moods ACF (array)
  image:       "https://...",           // featured image URL (nullable)
  hue:         220,                     // computed from category slug
  price:       "free",                  // "free" | "paid" (required)
  priceLabel:  "Free",                  // display string (required)
  critic:      true,                    // hfx_critic_pick ACF
  recurring:   "Every Sunday",          // _EventRecurrence (nullable)
  url:         "https://..."            // _EventURL / tickets link
}

Custom ACF fields needed.

These don't come from ECP — register them via Advanced Custom Fields on the tribe_events post type:

hfx_short_blurb (text, 90 char max) · hfx_editor_blurb (textarea) · hfx_critic_pick (true/false) · hfx_moods (checkbox: chill/rowdy/date/kids/solo/crew/free/rainy) · hfx_neighbourhood (select: 8 Halifax areas)

WordPress Setup

§ 10

Required plugins and settings before front-end work starts.

  • The Events Calendar Pro — main plugin. Enables recurring events, venue geocoding, REST API endpoints.
  • Event Aggregator — imports from Eventbrite, iCal, Meetup, Facebook, CSV. Set imported events to pending status so editors must approve before publish.
  • Advanced Custom Fields (ACF) — register all hfx_* fields on tribe_events post type. Export the field group as JSON for version control.
  • REST API endpoint — build /wp-json/hfx/v1/events returning normalized JSON per the contract in §08. Cache aggressively (60s transient minimum).
  • Category slugs locked — the 12 slugs in §04 must match exactly. The hue mapping is slug-based on the front-end. Renaming a slug breaks colour.
  • Venue geocoding on — ECP Pro setting. Required for map page lat/lng. Enable "Google Maps" under Events → Settings → Maps.
  • Moderation workflow — imported events and public submissions land in pending. Editor adds moods + neighbourhood + critic flag, then publishes.
  • Image size — add a custom image size at 800×600 (crop). Validate minimum dimensions on the submit form — images smaller than 400px wide degrade through the halftone.

Filter Logic

§ 11

The browse/filter system has 4 dimensions. All are AND logic — filters narrow, they don't expand.

// Quick filter (time-based)
tonight:  events where date === today
tomorrow: events where date === tomorrow
weekend:  events where date is Sat or Sun of current week
free:     events where price === "free"

// Category filter (multi-select, OR within dimension)
cats: ["music","comedy"] → events where category IN cats

// Mood filter (multi-select, OR within dimension)
moods: ["chill","date"] → events where event.mood intersects moods

// Neighbourhood filter (multi-select, OR within dimension)
hoods: ["North End","Downtown"] → events where hood IN hoods

// Combined: all 4 dimensions are AND'd together
filtered = events
  .filter(e => !quick || matchesQuick(e, quick))
  .filter(e => !cats.length || cats.includes(e.category))
  .filter(e => !moods.length || e.mood?.some(m => moods.includes(m)))
  .filter(e => !hoods.length || hoods.includes(e.hood))

Known Gotchas

§ 12
  • Container queries for responsive layout — the Broadsheet layout uses CSS @container queries, not @media queries. This means responsive behaviour triggers based on the parent container's width, not the viewport. Set container-type: inline-size on the page wrapper. This is intentional — it makes the layout work inside iframes and embedded contexts too.
  • oklch() colour function — the image treatment hue system uses oklch(). Supported in all modern browsers (Chrome 111+, Safari 15.4+, Firefox 113+). For older browser support, provide an rgb() fallback per category.
  • Leaflet tiles need internet — the CartoDB tile layer requires a network connection. The site doesn't work fully offline. For a PWA, consider caching tiles with a service worker.
  • Recurring events expand client-side — ECP's recurring events are one post with many occurrences. The heatmap and filter logic need to expand occurrences. Cache this computation — it's expensive on large event sets.
  • Mix-blend-mode on images needs black background — the halftone treatment uses mix-blend-mode: multiply. The container div must have background: #000 so the blend works correctly before the image loads. Missing this causes a white flash.
  • Playfair Display italic is a separate font — load ital,wght@1,700;1,900 in the Google Fonts URL. Missing the italic subset makes all italic text fall back to synthetic slant, which looks wrong.
  • Anton is all-caps by design — always pair with text-transform: uppercase. Lowercase Anton looks unintentional.
